# 02. 게임 시스템 설계 (Core Systems)

> 본 문서는 기획서의 **메인** 챕터다. 시스템 깊이와 데이터 구조까지 다룬다.

## 2.1 게임 루프 구조

```
[메인 화면(마을)]
   ├─ 캐릭터 선택
   ├─ 영구 업그레이드 (도토리 나무)
   ├─ 상점 (스킨/패스/캐릭터)
   └─ 던전 입장 ──▶ [한 런 (3~7분)]
                       ├─ 1층 ▶ 보상 선택(룬 카드 3택 1)
                       ├─ 2층 ▶ 보상 선택
                       ├─ 3층(엘리트) ▶ 강화 룬
                       ├─ 4층 ▶ 보상
                       ├─ 5층(보스) ▶ 도토리 + 룬
                       └─ 결과 화면 ▶ 재화/언락 ▶ 마을로 복귀
```

### 한 런(Run)의 흐름

| 단계 | 시간(분) | 내용 |
| --- | --- | --- |
| 입장·캐릭터 확인 | 0~0:10 | 던전 도착 컷, 자동 |
| 1층 (정찰) | 0:10~1:00 | 잡몹 6~8마리. 룬 카드 1장 픽 |
| 2층 (전투) | 1:00~2:00 | 잡몹 + 작은 함정. 룬 1장 |
| 3층 (엘리트 또는 휴식) | 2:00~3:30 | 엘리트 처치 시 강화 룬 / 휴식방은 회복+상점 |
| 4층 (위협) | 3:30~5:00 | 잡몹 밀집 + 미니보스 가능 |
| 5층 (보스) | 5:00~7:00 | 보스 1마리. 클리어 시 마법 도토리 1개 |
| 결과 | 7:00 | 정산. 부활 광고 슬롯은 보스전에서 죽었을 때만 |

**런 완료 보상**: 도토리(영구재화) + 도토리잎(메타재화) + 진행도

### 일일 루프 (DAU 견인)

- **데일리 미션 3개** (예: "토끼 영웅으로 1런 클리어", "엘리트 3회 처치", "도토리 5개 수집")
- **보상**: 도토리잎 + 100% 보너스 광고 옵션
- **체력 시스템 없음** (라이트 유저 친화·이탈 방지)

## 2.2 핵심 전투 시스템

### 조작

- **이동**: 화면 좌측 가상 조이스틱 (드래그)
- **자동 공격**: 캐릭터가 가장 가까운 적을 자동으로 공격 (Vampire Survivors / Brotato 스타일)
- **액티브 스킬**: 우측 하단 버튼 1~2개. 쿨다운 8~20초
- **회피/구르기 없음** (한 손 조작 단순화). 대신 **이동만 잘하면 모든 공격 회피 가능**한 디자인

### 전투 루트 — Top-down 2D, 화면 고정

- 카메라는 캐릭터 중심 추적
- 한 방(층)은 약 화면 1.5배 크기. 적을 모두 처치하면 다음 층으로 가는 포털 활성화
- 충돌은 사각형 AABB만 사용 (성능·구현 단순)

### 데미지 공식

```
최종 데미지 = (기본 공격력 × (1 + 공격력 보정 합산)) × 크리티컬 배율 × 속성 배율 - 방어력
```

| 변수 | 기본값 | 메모 |
| --- | --- | --- |
| 기본 공격력 | 캐릭터별 5~12 | 1레벨 기준 |
| 공격력 보정 | 0% | 룬·아이템 합산 |
| 크리티컬 배율 | 1.0 또는 1.5 | 크리 확률에 따라 |
| 속성 배율 | 0.8 / 1.0 / 1.3 | 약점·내성·일반 |
| 방어력 | 0~5 | 잡몹 0, 보스 5 |

> 데미지·체력 수치는 작게 유지(1~999 범위). 큰 숫자 인플레이션은 캐주얼 톤과 맞지 않음.

### 속성 시스템 (간단 3원소)

| 속성 | 강함 | 약함 |
| --- | --- | --- |
| 🌿 자연 | 🪨 대지 | 🔥 불 |
| 🔥 불 | 🌿 자연 | 💧 물 |
| 💧 물 | 🔥 불 | 🌿 자연 |
| 🪨 대지 | (중립) | (중립) — 보스 전용 속성 |

- 잡몹은 1속성, 보스는 1속성 + 2약점(강조 학습용)
- 캐릭터는 기본 1속성. 룬으로 다른 속성 공격 가능

### 액티브 스킬

- 캐릭터마다 고유 1개. 쿨다운식.
- 추가 스킬은 룬으로 획득 (최대 2 슬롯)

#### 예시 (밤 다람쥐 도리)
- **롤링 도토리**: 큰 도토리를 굴려 직선 관통, 8초 쿨다운, 데미지 200%

## 2.3 캐릭터 시스템

### 구조

각 캐릭터는 다음 데이터를 가진다:

```yaml
character:
  id: "doori"
  display_name: "밤 다람쥐 도리"
  unlock_method: "starter"      # starter / acorns / iap / quest
  unlock_cost: 0
  rarity: "common"              # common / rare / epic
  base_stats:
    hp: 80
    atk: 10
    move_speed: 110             # px/sec
    attack_speed: 1.2           # 초당 공격 횟수
    crit_rate: 0.05
    crit_damage: 1.5
  element: "nature"
  weapon_type: "ranged"         # melee / ranged / magic
  base_range: 120               # px
  passive: "도토리를 주울 때 회복 +1"
  active_skill_id: "rolling_acorn"
```

### 캐릭터 차별화 핵심

성능보다 **플레이 스타일 차이**로 차별화. 능력치는 비슷하게 균형 잡고, 다음 축에서 차이를 둠:

| 축 | 예시 |
| --- | --- |
| **공격 거리** | 근접 / 중거리 / 장거리 |
| **공격 패턴** | 단발 / 부채꼴 / 관통 / 추적 |
| **스킬 성격** | 공격형 / 회복형 / 군중제어형 |
| **속성** | 자연 / 불 / 물 / 대지 |
| **크기·이동속도** | 작고 빠름 / 크고 단단 |

### 영구 성장 — 도토리 나무

마을 중앙의 거대한 나무. 도토리(런 보상)를 투입하면 **모든 캐릭터에 적용되는 글로벌 패시브** 잠금 해제.

| 가지 | 노드 예시 | 비용(누적) |
| --- | --- | --- |
| 🌿 잎 (생존) | HP +5%, 시작 시 보호막, 회복 효율↑ | 50 → 200 → 500 |
| 🌰 열매 (자원) | 도토리잎 획득량+, 상점 할인 | 50 → 200 → 500 |
| 🌸 꽃 (전투) | 크리율+, 공격속도+, 시작 룬 추가 | 100 → 300 → 700 |

> 메타 성장은 **부드럽게**. 솔로 콘텐츠가 적으므로 노드 30개 정도로 한 달치 목표 부여.

## 2.4 룬 카드 시스템 (런 내 빌드)

매 층 클리어 시 **룬 카드 3장 중 1장**을 픽한다. 빌드의 핵심.

### 룬 카드 구조

```yaml
rune:
  id: "thorn_armor"
  name: "가시 도토리 갑옷"
  rarity: "common"   # common / rare / epic / legendary
  category: "defense" # offense / defense / utility / synergy
  description: "피격 시 적에게 데미지의 30%를 반사한다."
  effect:
    type: "thorn"
    value: 0.3
  stack_limit: 3      # 최대 3중첩
  prerequisites: []   # 시너지 카드는 선행 룬 필요
```

### 룬 카테고리 (목표: MVP에 30~40종)

| 카테고리 | 비율 | 예시 |
| --- | --- | --- |
| Offense (공격) | 35% | 공격력+, 공속+, 다중타격, 관통 |
| Defense (방어) | 25% | HP+, 보호막, 가시 반사, 회피율 |
| Utility (유틸) | 20% | 이동속도+, 도토리 자석, 시야 확장 |
| Synergy (시너지) | 20% | 특정 룬 보유 시 진화하는 카드 |

### 시너지 — "진화 룬"

특정 룬 3중첩 + 다른 특정 룬 1중첩 시 **합성 진화** 발생.
한 런에 2~3개의 진화 룬을 노리는 것이 빌드의 묘미.

#### 예시
- **불꽃 화살** ×3 + **바람의 깃털** ×1 → **유성우** (10초마다 화면 전체 폭격)
- **가시 갑옷** ×3 + **HP 증가** ×1 → **고슴도치 모드** (피격 시 가시 반사 + 무적 0.3초)

> MVP에는 진화 룬 6~8종이면 충분. 진화 발견 자체가 콘텐츠.

### 등장 확률 보정

레어 등장 확률은 **층이 깊을수록·언락된 룬이 많을수록** 상승. 시작 1~2층은 평범한 룬으로 학습 곡선 완만하게.

```
common  : 1층 70% → 5층 30%
rare    : 1층 25% → 5층 45%
epic    : 1층  5% → 5층 20%
legendary: 1층 0% → 5층  5%
```

## 2.5 던전 생성 시스템

### 절차적 생성 — "방 단위 시드"

완전 무작위 타일 생성은 솔로엔 무리. **방(Room) 프리팹 풀 + 시드 기반 조합**으로 절차성 확보.

```
한 던전 = 5층
한 층 = 1개의 방
한 방 = 프리팹 풀에서 1개 추첨 + 시드된 몬스터 스폰 패턴
```

### 방 프리팹 카테고리

| 타입 | MVP 개수 | 설명 |
| --- | --- | --- |
| 일반 전투방 | 12 | 잡몹 처치 |
| 엘리트방 | 4 | 강한 적 1~2 + 잡몹 |
| 휴식방 | 3 | 회복샘 + 상점 NPC |
| 보스방 | 4 | 던전별 1개씩 (4개 던전) |
| 비밀방(레어 출현) | 2 | 보너스 도토리 |

→ MVP는 방 25개. 한 사람이 1~2주에 만들 수 있는 양.

### 진행 시스템

- **던전 1: 도토리 숲** — 자연 속성, 5층, 보스: 비대해진 다람쥐 왕
- **던전 2: 불씨 동굴** — 불 속성, 7층, 보스: 용암 두꺼비 (해금: 도토리 30개)
- **던전 3: 이끼 늪지** — 물 속성, 9층, 보스: 거대 거머리 (해금: 도토리 80개)
- **던전 4: 고대 뿌리** — 대지 속성, 12층, 보스: 시들어가는 나무의 정 (해금: 도토리 200개)

> MVP는 **던전 1만 완성**. 던전 2는 출시 후 1차 업데이트.

## 2.6 데이터 주도 설계 원칙

솔로 개발에서 가장 중요. 모든 게임 데이터는 코드 밖 JSON/CSV에 있어야 한다.

```
res://data/
  characters.json
  runes.json
  enemies.json
  rooms.json
  drops.json
  meta_progression.json
  localization/
    ko.json
    en.json
```

밸런스 패치 = JSON 수정 + 재빌드. 코드 수정 0.

## 2.7 진행 저장 / 세이브

| 항목 | 저장 위치 | 비고 |
| --- | --- | --- |
| 캐릭터 언락, 메타 업그레이드, 재화 | `user://save.dat` (Godot user dir) | JSON, 솔트된 해시로 변조 방지 |
| 진행 중인 런 | 메모리만 | 앱 강제종료 시 런은 무효 (라이트 유저용) |
| 통계 (총 처치 수, 플레이 시간) | 동일 파일 | 업적 트리거 |
| 클라우드 세이브 | 미적용 (1차) | 시즌 2에 Play Games Services 도입 |

## 2.8 난이도 곡선 & 첫 30분

라이트 유저의 첫 30분이 잔류율을 결정한다.

| 분 | 목표 |
| --- | --- |
| 0~3분 | 첫 런 보장 클리어. 튜토리얼은 **인게임에 자연 통합**, 별도 강제 X |
| 3~10분 | 두 번째 런에서 첫 피드백(다른 룬 픽). 룬 시스템 학습 |
| 10~20분 | 첫 캐릭터 추가 언락 가능 (도토리 5개 = 한 캐릭터) |
| 20~30분 | 두 번째 던전 첫 도전 또는 메타 업그레이드 첫 노드 |

> **첫 30분에 "다음에 또 해야지" 동기 3개**(새 캐릭터, 새 룬, 새 던전 중 2개 이상)를 노출하는 것이 목표.
