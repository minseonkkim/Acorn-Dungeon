# 01. 개요 (Overview)

## 1.1 게임 컨셉

- **제목(가)**: 도토리 던전 (Acorn Dungeon)
- **장르**: 캐주얼 로그라이크 던전 RPG (자동 전투 + 수동 이동)
- **플랫폼**: Android 우선 출시 → 추후 iOS 확장
- **세션 길이**: 1런 3~7분, 1일 평균 4~6런 가정
- **조작**: 가상 조이스틱 1개 (왼손) + 액티브 스킬 1~2개 버튼 (오른손 엄지)

### 핵심 컨셉 문장

> 평화로운 동물 마을의 거대한 나무가 시들어간다. 마을의 영웅들이 뿌리 아래 던전으로 내려가 **마법의 도토리**를 모아 나무를 되살린다. 매번 던전은 다른 모습이고, 영웅이 얻는 능력도 매번 달라진다.

## 1.2 타겟 유저

- **1차 타겟**: 전 연령대 라이트 모바일 게이머. 동물의 숲, 슬라임 목장, 뱀파이어 서바이버 류를 즐기는 층.
- **2차 타겟**: 픽셀아트·인디 게임을 좋아하는 코어 유저(스팀 인디 팬). 입소문·SNS 공유 동력.
- **연령**: 10대 후반 ~ 40대. 가족 친화적 전체 이용가 등급 목표.

### 유저 페르소나

| 페르소나 | 특징 | 플레이 패턴 |
| --- | --- | --- |
| 출근길 직장인 | 25~35세, 짧은 자투리 시간 | 1일 5~8런, 광고 시청 거부감 낮음 |
| 학생 | 14~20세, SNS 공유 활발 | 캐릭터 수집·꾸미기 동기. 인앱결제 평균↓, 광고↑ |
| 캐주얼 부모 | 30~45세, 자녀와 함께 | 단순 조작 선호. 패스/스킨 결제 가능성 높음 |

## 1.3 플랫폼 & 기술 스택

### 추천 스택: **Godot 4 (GDScript)**

솔로 개발자 + 1~3개월 MVP + 2D 픽셀아트 + Android 출시라는 조건에 가장 적합.

| 항목 | 이유 |
| --- | --- |
| 2D 픽셀아트 워크플로우 | TileMap·Sprite2D·Camera2D가 픽셀 퍼펙트에 최적화. `snap_2d_transforms_to_pixel` 옵션 한 줄로 해결 |
| 학습 곡선 | GDScript는 Python 유사. 솔로 개발 학습 시간 최소화 |
| 빌드 크기 | 안드로이드 APK 약 30~50MB. 라이트 유저에게 다운로드 진입장벽↓ |
| 라이선스 | MIT, 매출 로열티 0원 |
| Android Export | 공식 가이드 잘 정비됨. AdMob·IAP은 서드파티 플러그인 사용 |
| 에디터 자체 무게 | 100MB 이하. 노트북 개발 친화 |

### 대안 비교

| 스택 | 장점 | 단점 | 결론 |
| --- | --- | --- | --- |
| **Unity** | 에셋 스토어, AdMob/IAP 공식 플러그인, 자료 압도적 | 빌드 무거움, 라이선스 정책 변동 이력, 솔로엔 오버킬 | 차선 |
| **Flutter/Flame** | 앱 개발 경험자에 친숙 | 게임 도구 빈약. 추후 확장 어려움 | 비추 |
| **React Native + 게임 엔진** | 웹뷰 확장 쉬움 | 성능·픽셀퍼펙트 어려움 | 비추 |

> **결정**: Godot 4 채택. (이미 Unity 경험이 있다면 Unity로 가도 무방하지만, 본 기획서는 Godot 기준으로 작성)

### 외부 서비스

- **광고**: AdMob (Google) — 보상형 동영상이 핵심
- **인앱결제**: Google Play Billing
- **분석**: Firebase Analytics + Crashlytics (무료, 솔로에 충분)
- **백엔드**: 없음. **MVP는 100% 오프라인** (서버 비용·운영 부담 제거)
- **클라우드 세이브**: Google Play Games Services 사용 (추후 시즌 2)

## 1.4 아트 방향

### 비주얼 톤

- **따뜻하고 귀여운 동물·판타지** (스튜디오 지브리 + 슬라임 랜처 + 도토리 같은 자연물 모티프)
- 스타일: **카툰 일러스트 풍**(부드러운 셰이딩, 안티에일리어싱 허용). 정통 도트 픽셀아트 X
- 컬러 팔레트: 채도 중간, 명도 높음. 어두운 던전에서도 **포근한 톤**을 유지. 32색 강제 제한 없음
- 톤 레퍼런스: *Slime Rancher*의 둥근 귀여움 + *Don't Starve*의 동물 캐릭터 표현 + *Hades*의 동선 명료성

### 스프라이트 사양

| 항목 | 값 |
| --- | --- |
| 프레임당 픽셀 (시트 원본) | **128x128 픽셀** |
| 시트 사양 | **3072x128 PNG, 1행 × 24열 (가로 strip)** |
| 게임 내 표시 크기 | **96x96** (`AnimatedSprite2D` scale = 0.75) |
| 시트 레이아웃 | 24컷 = idle(4) + walk(4) + attack(4) + hit(4) + skill(4) + death(4) |
| 액션별 프레임 수 | 모든 액션 **4프레임 통일** (8fps 기준 0.5초 사이클) |
| 타깃 해상도 | 360x640 가상 → 디바이스 해상도로 정수 배율 스케일. 캐릭터 96px = 화면 폭 약 1/3.75 |
| 애니메이션 FPS | 8 fps |
| 방향 처리 | 정면 1세트 + 코드 `flip_h`로 좌우 처리 (4방향 X) |
| 비주얼 톤 | **카툰 일러스트 풍** (AI 생성 매끄러운 일러스트 기준, 안티에일리어싱·그라데이션 셰이딩 허용) |

> **솔로 작업 팁**: 모든 캐릭터·적이 동일한 128px × 24컷 가로 strip 레이아웃을 따르면 `AnimatedSprite2D` + `SpriteFrames` 리소스 하나의 템플릿으로 전 캐릭터를 처리할 수 있다. AI 생성 시트를 다운스케일하지 않고 원본 그대로 사용하므로 별도 이미지 변환 단계가 불필요하다.

### UI 톤

- 둥근 모서리, 두꺼운 테두리(2px)
- 폰트: Galmuri11 (한글 픽셀 비트맵, 무료 상용 가능)
- 사운드: 8bit~chiptune 보다는 **잔잔한 ukulele/celesta** 계열로 따뜻함 강조

## 1.5 비-목표 (Non-Goals)

명시적으로 **하지 않는 것**들. 솔로 MVP 스코프 보호.

- 멀티플레이·실시간 PvP
- 길드·소셜 시스템
- 3D 그래픽 어떤 요소도
- 실시간 음성·채팅
- 시나리오 분기·풀보이스 컷씬
- iOS 동시 출시 (안드로이드 안정화 후)
- 자체 백엔드 서버
